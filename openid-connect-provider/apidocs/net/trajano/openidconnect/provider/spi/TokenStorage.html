        Store the token response in the storage.

        It should use the TokenResponse.getAccessToken() and TokenResponse.getRefreshToken() to store the mapping for the access token and refresh token to the id token respectively. The splitting up logic was not done in TokenProvider to allow implementations of the TokenStorage that support composite key building.

 refreshTokenToTokenResponse.put(idTokenResponse.getRefreshToken(), idTokenResponse);

      • store

        void store(net.trajano.openidconnect.token.IdToken idToken,
         net.trajano.openidconnect.token.IdTokenResponse idTokenResponse,
         String code,
         javax.json.JsonObject claims)

        Store the token response in the storage and add a code mapping for it.

        It should use the TokenResponse.getAccessToken() and TokenResponse.getRefreshToken() to store the mapping for the access token and refresh token to the id token respectively. The splitting up logic was not done in TokenProvider to allow implementations of the TokenStorage that support composite key building.

        This would generally call #store(IdTokenResponse) first followed by storing the code mapping.

        Parameters:
        idToken - id token
        idTokenResponse - token response to store
        code - code
        claims - claims requested
